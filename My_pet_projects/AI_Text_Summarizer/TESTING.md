# Инструкция по тестированию

## Тестирование Ollama

### 1. Проверка работы Ollama Helper

Запустите тестовый скрипт:

```bash
python test_ollama.py
```

Скрипт проверит:
- ✅ Запущена ли Ollama
- ✅ Какие модели установлены локально
- ✅ Какие модели доступны для скачивания

### 2. Тестирование основного приложения

#### Шаг 1: Запуск приложения

```bash
python main.py
```

#### Шаг 2: Проверка автозапуска Ollama

Если Ollama не запущена:
- ✅ Должно появиться окно с предложением запустить Ollama
- ✅ При согласии Ollama должна запуститься автоматически
- ✅ Список моделей должен обновиться

#### Шаг 3: Проверка списка моделей

В выпадающем списке моделей должны отображаться:

1. **Заголовок "--- Локальные модели ---"** (если есть установленные модели)
2. **Локальные модели с иконкой 📦** (например: `📦 llama3.2:latest`)
3. **Заголовок "--- Доступные для скачивания ---"**
4. **Облачные модели с иконкой ☁️** (например: `☁️ llama3.2:latest`)

#### Шаг 4: Проверка кнопки обновления

- ✅ Нажмите кнопку 🔄 рядом со списком моделей
- ✅ Список должен обновиться

#### Шаг 5: Проверка скачивания модели

1. Выберите облачную модель (☁️)
2. Нажмите "Summarize"
3. Должно появиться окно с предложением скачать модель
4. При согласии начнется скачивание с прогресс-баром

#### Шаг 6: Проверка суммаризации

1. Нажмите "Open File" и выберите документ (PDF, DOCX или TXT)
2. Выберите локальную модель (📦)
3. Выберите режим (bullet/outline/executive)
4. Выберите стиль (technical/legal/scientific)
5. Нажмите "Summarize"
6. Результат должен появиться в правой панели

## Отладка

### Проблема: Локальные модели не отображаются

**Решение:**
1. Убедитесь, что Ollama запущена: `ollama serve`
2. Проверьте список моделей: `ollama list`
3. Нажмите кнопку 🔄 для обновления списка
4. Проверьте консоль на наличие сообщений `DEBUG:`

### Проблема: Облачные модели не отображаются

**Решение:**
1. Проверьте файл `core/ollama_helper.py`, метод `get_cloud_models()`
2. Убедитесь, что список `popular_models` не пустой
3. Проверьте консоль на наличие сообщений `DEBUG:`

### Проблема: Ollama не запускается автоматически

**Решение:**
1. Убедитесь, что Ollama установлена: `ollama --version`
2. Запустите Ollama вручную: `ollama serve`
3. Проверьте, что Ollama доступна на `http://localhost:11434`

## Ожидаемый вывод в консоли

При запуске приложения в консоли должны появиться сообщения:

```
DEBUG: Локальные модели Ollama: ['llama3.2:latest', 'mistral:latest']
DEBUG: Добавлена локальная модель: llama3.2:latest
DEBUG: Добавлена локальная модель: mistral:latest
DEBUG: Облачные модели: 12 шт.
DEBUG: Добавлена облачная модель: llama3.2:latest
DEBUG: Добавлена облачная модель: llama3.1:latest
...
```

Если сообщения не появляются, проверьте:
- Запущена ли Ollama
- Правильно ли работает метод `get_local_models()`
